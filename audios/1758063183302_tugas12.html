1. Pengenalan NTP (Network Time Protocol)

Menurut dokumentasi MikroTik, NTP (Network Time Protocol) adalah sebuah layanan jaringan yang berfungsi untuk menjaga keseragaman waktu pada semua perangkat jaringan. NTP bekerja dengan cara menyesuaikan jam sistem perangkat terhadap waktu global yang bersumber dari referensi terpercaya, seperti NTP server publik atau GPS clock. Sinkronisasi waktu ini sangat krusial karena memengaruhi banyak aspek, misalnya keakuratan pencatatan log aktivitas jaringan, konsistensi dalam pengaturan jadwal otomatis, hingga penerapan kebijakan keamanan yang berbasis perbedaan waktu.

Menurut Tech@Layer-x, NTP merupakan protokol standar yang umum dipakai pada perangkat jaringan seperti router, switch, server, hingga komputer client. Tanpa adanya NTP, jam internal perangkat dapat mengalami time drift atau pergeseran waktu secara bertahap, yang menimbulkan masalah serius, seperti ketidaksesuaian catatan log, kegagalan autentikasi berbasis waktu, maupun gangguan komunikasi antar server yang membutuhkan presisi waktu tinggi. Dengan memanfaatkan NTP, perangkat dapat memperoleh referensi waktu yang stabil dari sumber terpercaya, sehingga semua sistem dalam jaringan tetap berjalan dengan sinkron.

Dari beberapa pendapat tersebut dapat disimpulkan bahwa NTP adalah protokol fundamental yang menjamin kesamaan waktu di seluruh perangkat jaringan. Dengan adanya sinkronisasi waktu, administrator dapat memastikan keakuratan log aktivitas, keberhasilan autentikasi berbasis waktu, serta stabilitas komunikasi antar perangkat yang membutuhkan ketepatan waktu tinggi.

2. Mengonfigurasi Dasar NTP Client

Menurut dokumentasi resmi MikroTik, pada RouterOS versi 7, konfigurasi NTP Client dapat dilakukan dengan cara mengaktifkan parameter enabled=yes, kemudian menambahkan server sinkronisasi waktu, baik berupa alamat IP maupun DNS. RouterOS juga sudah mendukung fitur failover otomatis, sehingga apabila server utama tidak merespons, sistem akan otomatis berpindah ke server cadangan.

Menurut Tech@Layer-x, langkah awal dalam konfigurasi NTP Client adalah dengan menentukan zona waktu sesuai lokasi perangkat agar hasil sinkronisasi sesuai dengan waktu lokal. Setelah itu, NTP Client perlu diaktifkan, kemudian ditambahkan daftar server NTP menggunakan perintah /system ntp client servers add, sehingga router memiliki lebih dari satu sumber sinkronisasi untuk menjaga akurasi waktu.

Dari beberapa pendapat tersebut dapat disimpulkan bahwa konfigurasi NTP Client sebaiknya dimulai dengan mengatur zona waktu perangkat, lalu mengaktifkan layanan NTP Client, dan menambahkan lebih dari satu server agar sinkronisasi waktu berjalan stabil, akurat, serta tetap berfungsi meskipun salah satu server tidak tersedia.

Langkah-Langkah Konfigurasi NTP Client

1. Mengaktifkan NTP Client dan Menentukan Server Utama & Cadangan
Masukkan perintah berikut pada CLI:
/system ntp client set enabled=yes primary-ntp=203.160.128.59 secondary-ntp=203.89.200.1

enabled=yes → mengaktifkan layanan NTP Client
primary-ntp → server utama yang digunakan untuk sinkronisasi waktu
secondary-ntp → server cadangan jika server utama tidak merespons

2. Mengecek Status Konfigurasi NTP Client
Gunakan perintah berikut untuk memastikan bahwa konfigurasi sudah berjalan dengan benar:
/system ntp client print

Dari hasil perintah ini akan terlihat apakah NTP Client sudah aktif, serta server mana yang sedang digunakan untuk sinkronisasi waktu.

3. Konfigurasi Time Zone dan Format Waktu

Menurut sumber Microsoft, pengaturan zona waktu dan format waktu pada Windows 10 maupun Windows 11 dapat dilakukan melalui menu Pengaturan → Waktu & Bahasa → Tanggal & Waktu. Di sana, pengguna dapat memilih opsi "Atur waktu secara otomatis" atau "Atur zona waktu secara otomatis". Jika ingin melakukan pengaturan secara manual, opsi otomatis bisa dinonaktifkan, lalu pengguna dapat mengubah tanggal dan waktu secara langsung melalui tombol Ubah atau memilih zona waktu yang sesuai dari menu dropdown. Selain itu, Windows juga menyediakan pengaturan tambahan seperti menyesuaikan waktu musim panas (daylight saving time), menampilkan detik pada system tray, serta menentukan format tanggal dan waktu yang ditampilkan. Dengan demikian, pengguna bisa menyesuaikan tampilan jam sesuai kebutuhan.

Menurut tutorial dari Hostinger, di sistem operasi Linux Ubuntu terdapat dua cara utama dalam mengatur zona waktu, yaitu menggunakan antarmuka grafis (GUI) maupun command-line (CLI/terminal). Jika melalui GUI, pengguna cukup membuka Settings → Date & Time, lalu menonaktifkan opsi zona waktu otomatis. Setelah itu, zona waktu bisa dipilih secara manual melalui peta interaktif atau fitur pencarian. Sementara itu, untuk pengguna yang lebih terbiasa dengan terminal, tersedia dua metode: pertama, menggunakan tzselect untuk perubahan sementara (akan hilang setelah sistem direstart); kedua, dengan timedatectl untuk konfigurasi permanen. Misalnya, menjalankan perintah sudo timedatectl set-timezone Asia/Jakarta akan langsung mengubah zona waktu sistem ke WIB. Untuk memastikan hasilnya, pengguna bisa mengetikkan perintah timedatectl untuk melihat status konfigurasi waktu saat ini.

Dari beberapa pendapat tersebut dapat disimpulkan bahwa pengaturan zona waktu dan format waktu dapat dilakukan dengan mudah, baik melalui antarmuka grafis maupun command-line. Pemilihan metode tentu tergantung sistem operasi dan preferensi pengguna, namun keduanya sama-sama sederhana, praktis, dan cepat.

Langkah Konfigurasi Zona Waktu di MikroTik (via CLI):

1. Jalankan perintah untuk masuk ke menu pengaturan waktu:
/system clock

2. Atur zona waktu sesuai lokasi, misalnya untuk Jakarta (WIB):
set time-zone-name=Asia/Jakarta

3. Cek hasil konfigurasi apakah sudah sesuai:
/system clock print

Dengan langkah ini, router MikroTik akan menampilkan waktu lokal yang benar, sehingga log dan jadwal (scheduler) bisa sinkron sesuai zona waktu setempat.

4. Troubleshooting

Menurut dokumentasi MikroTik, proses troubleshooting NTP (Network Time Protocol) biasanya dimulai dengan memeriksa apakah router masih bisa menjangkau server NTP yang ditentukan. Jika konektivitas bermasalah, langkah berikutnya adalah mengecek konfigurasi DNS untuk memastikan nama domain server NTP bisa diresolusi dengan benar. Jika server utama tidak merespons, pengguna disarankan mengganti atau menambahkan server lain sebagai cadangan agar sinkronisasi waktu tetap berjalan.

Menurut Tech@Layer-x, proses uji konektivitas bisa dilakukan dengan perintah /tool ping langsung ke server NTP, baik menggunakan alamat IP maupun domain. Setelah itu, DNS perlu dipastikan berfungsi normal dengan cara melakukan ping ke nama domain umum, misalnya google.com. Selain itu, penting juga menambahkan server cadangan (secondary NTP server) agar sinkronisasi waktu tidak terganggu jika server utama gagal diakses.

Dari beberapa pendapat tersebut, dapat disimpulkan bahwa troubleshooting NTP pada MikroTik bukan hanya memeriksa konektivitas internet, tetapi juga memastikan DNS berjalan baik, serta menambahkan server cadangan untuk menjaga kestabilan sinkronisasi waktu.

Langkah-Langkah Troubleshooting NTP di MikroTik

1. Cek konektivitas internet
/tool ping 8.8.8.8

Jika ada reply, artinya koneksi internet normal.
Jika request timeout, berarti ada masalah pada koneksi fisik atau konfigurasi gateway.

2. Cek DNS
/ip dns print
/tool ping google.com

Pastikan DNS server sudah terkonfigurasi.
Jika domain bisa diping, berarti DNS berjalan dengan baik.
Jika gagal, periksa pengaturan DNS atau ganti dengan DNS publik (misalnya 8.8.8.8 atau 1.1.1.1).

3. Tambahkan server NTP cadangan
/system ntp client set enabled=yes primary-ntp=203.160.128.3 secondary-ntp=202.162.32.12

primary-ntp = server utama.
secondary-ntp = server cadangan, dipakai otomatis jika server utama gagal.

4. Verifikasi status NTP client
/system ntp client print

Untuk RouterOS v6, cukup melihat status pada menu print.
Untuk RouterOS v7, bisa lebih detail dengan:
/system ntp client status

→ Akan terlihat apakah sinkronisasi berhasil, jam sudah sesuai, dan ke server mana router terhubung.

5. Konfigurasi NTP Server

Menurut dokumentasi MikroTik, selain berfungsi sebagai client, perangkat MikroTik juga dapat dikonfigurasi sebagai NTP Server untuk menyediakan layanan sinkronisasi waktu bagi perangkat lain dalam jaringan lokal. Dengan mengaktifkan mode server, router dapat memberikan waktu yang akurat kepada client-client di jaringan internal, sehingga semua perangkat memiliki waktu yang seragam.

Menurut Tech@Layer-x, konfigurasi NTP Server pada MikroTik memungkinkan administrator untuk membuat sumber waktu lokal yang dapat diandalkan, terutama jika koneksi internet terputus. Server NTP lokal ini akan tetap menyediakan waktu yang relatif akurat meskipun tidak dapat melakukan sinkronisasi dengan server NTP publik.

Dari beberapa pendapat tersebut dapat disimpulkan bahwa konfigurasi NTP Server pada MikroTik memberikan fleksibilitas dalam mengelola waktu jaringan, memastikan konsistensi waktu bahkan dalam kondisi koneksi internet terbatas, serta mengurangi ketergantungan pada server eksternal.

Langkah-Langkah Konfigurasi NTP Server di MikroTik:

1. Aktifkan mode NTP Server:
/system ntp server set enabled=yes

2. Tentukan interface yang akan melayani request NTP:
/system ntp server set interface=ether1

3. Verifikasi status NTP Server:
/system ntp server print

Dengan konfigurasi ini, perangkat MikroTik akan berfungsi sebagai NTP Server yang dapat diakses oleh client dalam jaringan lokal.

6. Monitoring dan Maintenance NTP

Menurut dokumentasi MikroTik, monitoring status NTP client dan server dapat dilakukan dengan perintah /system ntp client monitor dan /system ntp server monitor. Perintah ini memberikan informasi detail tentang status sinkronisasi, server yang digunakan, offset waktu, dan informasi lainnya yang berguna untuk memastikan NTP berfungsi dengan baik.

Menurut Tech@Layer-x, maintenance rutin pada konfigurasi NTP meliputi pemantauan log sistem untuk mendeteksi masalah sinkronisasi, memperbarui daftar server NTP jika diperlukan, serta memastikan bahwa zona waktu tetap sesuai dengan lokasi geografis perangkat.

Dari beberapa pendapat tersebut dapat disimpulkan bahwa monitoring dan maintenance rutin pada konfigurasi NTP sangat penting untuk memastikan akurasi waktu dan keandalan sistem. Dengan pemantauan yang teratur, administrator dapat mendeteksi dan menyelesaikan masalah sebelum berdampak pada operasional jaringan.

Langkah-Langkah Monitoring NTP:

1. Monitor status NTP Client:
/system ntp client monitor

2. Monitor status NTP Server:
/system ntp server monitor

3. Periksa log sistem untuk masalah NTP:
/log print where message~"NTP"

Dengan langkah-langkah ini, administrator dapat memastikan bahwa layanan NTP berjalan optimal dan melakukan tindakan korektif jika diperlukan.
